# 복리 엔지니어링 (Compounding Engineering) — 세션이 끝날 때마다 시스템이 똑똑해진다

> AI와 일하는 사람들 사이에서 자리 잡은 작업 방식입니다: **일을 끝내는 데서 멈추지 않고,
> 그 일에서 배운 것을 시스템에 침전시켜 다음 일이 더 쉬워지게 만든다.**

## 두 가지 일하는 방식

| | 선형 작업 | 복리 작업 |
|---|---|---|
| 세션이 끝나면 남는 것 | 산출물 하나 | 산출물 + **기준** + **도구** |
| 10번째 세션의 난이도 | 1번째와 같음 | 1번째보다 훨씬 쉬움 |
| 비유 | 단리 — 원금만 쌓임 | **복리 — 이자가 이자를 낳음** |

같은 5주를 보내도, 세션마다 기준과 도구가 침전된 사람의 작업실은 마지막 주에 전혀 다른 물건이 되어 있습니다. **이 프로그램의 산출물이 "26명이 다르게 자란 네 파일"인 이유가 이것입니다.**

## 이 작업실은 이미 복리 구조입니다

| 파일 | 침전되는 것 |
|---|---|
| `CLAUDE.md` | **기준** — "나는 어떻게 판단하는 사람인가"가 한 줄씩 쌓임 |
| `.claude/skills/` | **도구** — 반복하던 일이 스킬로 굳음 |
| `context.md` | **판단 상태** — 무엇을 왜 보고 있는지 |
| `journey.md` | **궤적** — 어디를 지나왔고 다음이 어디인지 |

그릇은 이미 있습니다. 문제는 **적립이 자동으로 일어나지 않는다**는 것 — 세션이 그냥 끝나면 이자는 붙지 않습니다.

## session-wrap의 의도 ①

`/session-wrap`의 세 줄(기준/판단/보강)은 **복리 이자를 실제로 적립하는 순간**입니다. 세션마다 3줄 — 5주면 작지 않은 원장이 됩니다. 투자에서 그렇듯, 복리의 조건은 수익률이 아니라 **거르지 않는 것**입니다.
