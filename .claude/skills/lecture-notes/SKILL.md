# lecture-notes — 강연 녹음 → 대본 + 요약

다운로드 폴더에 올린 강연·수업 m4a를 대본과 요약으로 바꿔 `outputs/`에 쌓는다. 변환은 로컬 whisper로 기기 안에서 이뤄지므로 음성이 외부 서버로 나가지 않는다.

## 처음 한 번: 환경 준비

`scripts/.venv`가 없으면(스킬을 새로 받았거나 새 기기에서 처음 쓰는 경우) 먼저 실행한다:

```
bash scripts/setup.sh
```

mlx-whisper와 ffmpeg(정적 바이너리)를 이 스킬 전용 가상환경에 설치한다 — 시스템 파이썬이나 brew를 건드리지 않는다. 첫 실행 시 whisper 모델(~1.5GB)을 자동으로 내려받으므로 몇 분 걸릴 수 있다. 이후로는 바로 넘어간다.

## 절차

1. **대상 파일 찾기**

   ```
   bash scripts/latest_recording.sh
   ```

   `~/Downloads`에서 가장 최근 수정된 `.m4a` 하나를 찾는다. 다운로드 폴더에 여러 개가 쌓여 있어도 가장 최근 것 하나만 처리한다. 없으면 사용자에게 파일을 올렸는지 확인한다.

2. **어떤 강연인지 확인한다**

   파일명(예: `관악로6길 6.m4a`)은 대개 녹음 위치일 뿐 강연 제목이 아니다. `curriculum.md`의 "AI Intensive 일정" 표에서 오늘 날짜에 해당하는 강연을 찾아 제목·주차 후보를 잡는다. 확실하지 않으면 짧게 물어본다: "이 녹음, 무슨 강연인가요? (제목·강연자)" 사용자가 말한 제목·강연자를 그대로 쓴다 — 대필 금지. 끝내 모르면 `unknown`으로 남긴다.

3. **변환한다**

   ```
   scripts/.venv/bin/python scripts/transcribe.py "<m4a 경로>"
   ```

   100분짜리 녹음 기준 로컬에서 수 분 걸린다. 대화를 막지 않도록 background로 돌리고, 진행 중이라고 사용자에게 알린다.

4. **요약을 얹는다** (대본 위에 구조화 — 대본을 요약으로 대체하지 않는다)

   - 대본 전체를 문서에 원문 그대로 포함한다. 편집·축약하지 않고 보존한다.
   - 그 위에 요약을 얹는다: 핵심 논점 3~5개, 강연에서 언급된 구체적 수치·사실.
   - CLAUDE.md의 출처 라벨 규칙을 그대로 따른다: 화자가 강연에서 직접 말한 수치·주장은 `[1차]`(그날 강연 발화)로 표시한다. Claude가 대본에 없는 내용을 추론·보충했다면 `[추정]`으로 분리한다. 대본만으로 확인 안 되는 부분은 `unknown`으로 두고 확인 방법을 제안한다.

5. **주차를 판단한다**

   `curriculum.md`의 일정표와 오늘 날짜를 대조해 몇 주차 강연인지 정한다. 애매하면 사용자에게 확인한다.

6. **저장한다**

   `outputs/week-0N-lecture-<slug>.md`에 저장한다. `<slug>`는 강연 제목을 영문·로마자로 짧게 줄인 것(예: `week2-lecture-vc-dealsourcing.md`). 문서 상단에 메타 정보(날짜, 강연 제목, 강연자, 원본 파일명 — 모르면 `unknown`)를 남긴다. 쓰기 전에 파일명과 요약 개요를 보여주고 확인받는다.

7. **원본을 정리한다**

   변환이 끝나면 원본 m4a를 `sources/lectures/<slug>.m4a`로 옮긴다 — 다운로드 폴더에 남겨두면 다음 실행 때 같은 파일을 다시 "가장 최근 파일"로 잡아 중복 처리할 수 있다. `sources/lectures/*.m4a`는 `.gitignore`에 있어 레포에 커밋되지 않는다 — 대본(텍스트)만 레포의 기록으로 남는다.

## 규칙

- 투자 조언 생성 금지 — 이 스킬은 강연 내용을 기록·구조화할 뿐 판단하지 않는다 (CLAUDE.md 공통 규칙).
- 음성 파일은 기기 밖으로 나가지 않는다 — 로컬 whisper만 쓰고 클라우드 API를 호출하지 않는다.
- 대본은 절대 요약으로 대체하지 않는다 — 항상 원문 그대로 함께 보존한다.
- API 키·크레덴셜이 필요 없다.
